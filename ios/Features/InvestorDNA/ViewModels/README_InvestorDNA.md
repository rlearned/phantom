# Investor DNA — API Contract

This document describes the API contract between the iOS frontend and the backend
for the Investor DNA feature. The frontend does **not** send ghost data to the backend;
the backend fetches the user's ghost trades directly from DynamoDB using the authenticated
`userId` extracted from the JWT.

---

## Endpoint

```
GET /v1/investor-dna
```

**Auth:** Bearer token (same Cognito JWT used by all other endpoints).  
**Request body:** None.  
**Query params:** None.

The backend fetches the 50 most recent ghost trades for the authenticated user, computes
the 6 behavioral scores, and generates a personalized insight string for each metric.

---

## Response JSON

```json
{
  "scores": {
    "intensity":    4,
    "momentum":     3,
    "conviction":   2,
    "caution":      5,
    "deliberation": 4,
    "sensitivity":  3
  },
  "insights": {
    "intensity":    "Over 70% of your ghost trades happen during volatile moments, suggesting emotional reactivity is a key driver of your hesitation.",
    "momentum":     "You tend to follow trending assets but hesitate mid-entry, showing momentum awareness without full commitment.",
    "conviction":   "You frequently second-guess your setups even when signals align, indicating low self-trust in your own analysis.",
    "caution":      "Most of your ghosts involve concerns about position sizing or capital loss — you are a natural risk manager.",
    "deliberation": "You over-analyze before acting, often re-checking your thesis multiple times and missing entry windows.",
    "sensitivity":  "News events and macro shifts heavily influence your decision to ghost, showing high market sentiment sensitivity."
  },
  "ghostsAnalyzed": 47
}
```

---

## Field Descriptions

### `scores` object

Each score is an **integer from 0 to 5** (inclusive), where:
- `0` = trait is essentially absent
- `5` = trait is at maximum intensity

| Field          | What it measures                                                                 |
|----------------|----------------------------------------------------------------------------------|
| `intensity`    | Emotional reactivity — how often the user ghosts during high-volatility moments  |
| `momentum`     | Trend-following bias — long vs. short ratio, following vs. fading moves          |
| `conviction`   | Self-trust in own analysis — inverse of second-guessing and doubt-related tags   |
| `caution`      | Risk aversion — frequency of position-sizing and capital-protection hesitations  |
| `deliberation` | Over-analysis — frequency of research/waiting/confirmation-seeking tags          |
| `sensitivity`  | External signal reactivity — influence of news, earnings, macro, sentiment tags  |

### `insights` object

Each value is a **personalized, plain-English description** of the user's specific score for
that metric. Max ~100 words. Written in second-person ("You tend to…"). Generated by the
backend based on the user's actual hesitation tag frequency and ghost patterns.

### `ghostsAnalyzed`

Integer. The number of ghost trades the backend actually used to compute the scores.
This may be less than 50 if the user has fewer ghosts on record.

The frontend uses this value to:
- Determine the **view state** (empty = 0, inProgress = 1–6, filled = 7+)
- Display "based on X ghost trades" in the page header subtitle

## iOS Integration Notes

When wiring up `InvestorDNAViewModel.loadData()`:

1. Call `GET /v1/investor-dna` (add `getInvestorDNA()` to `APIClient`).
2. Decode the response into an `InvestorDNAResponse` Codable struct.
3. Map `response.scores.*` → the 6 `@Published` score vars:
   ```swift
   intensityScore    = response.scores.intensity
   momentumScore     = response.scores.momentum
   convictionScore   = response.scores.conviction
   cautionScore      = response.scores.caution
   deliberationScore = response.scores.deliberation
   sensitivityScore  = response.scores.sensitivity
   ```
4. Store `response.insights.*` so each `TraitInfo.meaning` can be overridden with the
   backend-generated personalized string instead of the static fallback text.
5. Set `ghostCount` from `response.ghostsAnalyzed` (or keep the existing ghost list
   fetch for the "In your ghosts" section; both can run in parallel).

---

## Backend Implementation Notes

The backend should:

1. Call `appRepository.listGhosts(userId, 50)` — this already exists in `AppRepository.java`.
2. Extract `hesitationTags` and `direction` from each ghost.
3. Apply scoring heuristics (a reference implementation already exists in
   `InvestorDNAViewModel.computeMetricsLocally()` on the iOS side).
4. Generate the `insights` strings — either via hardcoded thresholds or an LLM call
   using the tag frequency summary as the prompt context.
5. Return the response JSON above.

New backend files needed:
- `service/InvestorDNAService.java`
- `controller/InvestorDNAController.java`
- Register route in `ApiHandler.java`: `GET /v1/investor-dna`
